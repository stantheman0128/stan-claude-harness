【GTC 2026：黃仁勳宣告 SaaS 將全面進化為 AaaS】
⠀⠀⠀
三月中旬，我坐在聖荷西 SAP Center 的觀眾席上，看著 NVIDIA 執行長黃仁勳切換到一張對比圖（圖二）。左半邊是熟悉的舊世界：一棟倉庫形狀的資料中心，上方連著兩個方塊——軟體和 SaaS、以及系統整合商（GSI）。右半邊是新世界：同樣的位置，卻多出了兩個從來不存在的角色——AI Providers 和 AI Natives。整張投影片的標題只有一行：「Enterprise IT Renaissance from SaaS to Agent-as-a-Service」。

那個瞬間我意識到，這張圖說的不只是 SaaS 要轉型，而是競技場突然多了兩類你從未預期的對手。這兩類對手不是從外部殺進來的，而是被底層平台直接孵化出來的。如果你是軟體新創的創辦人，這件事值得你停下來想一分鐘。
⠀⠀⠀
▐  OpenClaw 出現，標準正在快速收斂

要理解這場變化的速度，可以看 Jensen 在 keynote 展示的另一張圖（圖三）。奧地利開發者 Peter Steinberger 在 2026 年 1 月 25 日發布了開源專案 OpenClaw——一個讓 AI 模型從「對話工具」變成「可以自主執行任務的 Agent」的框架。它在數週內累積超過 30 萬個 GitHub stars，超越了 Linux 三十年的積累、React 的成長曲線，成為人類歷史上成長最快的開源專案。

Jensen 在台上說：「就像 Linux 給了這個世界一個共同的開源基礎，OpenClaw 正在對 Agentic AI 做同一件事。今天每一家公司都需要有自己的 OpenClaw 策略。」他把它比作 HTML 發明了網際網路時代、Kubernetes 定義了行動雲端時代的地位。NVIDIA 同步推出企業級版本 NemoClaw，加入安全護欄和政策引擎，讓企業可以在內部私有環境部署 Agent，而不必擔心敏感資料外洩。

這件事對 SaaS 新創的意義在於：當 Agentic AI 的標準確立，整個軟體的使用方式就會重寫。Agent 不再需要透過精美的 UI 操作你的產品，它可以直接呼叫工具、讀取檔案、跨系統執行任務。你花多年時間打磨的操作介面，正在從護城河變成可被繞過的障礙。
⠀⠀⠀
▐  平台往上長，競技場的地圖改了

要理解這場重組的結構，Jensen 的「AI 五層蛋糕」框架提供了清晰的地圖：從最底層的土地與能源、基礎設施、晶片，到第四層的平台（CUDA、OpenClaw、NeMo），最上層才是模型與應用。台灣軟體新創長期活在最上層的應用層，但這次的威脅並非來自同層的競爭者——而是第四層平台正在主動向上長，直接孵化出新型態的玩家。

這就是為什麼那張投影片的右半邊，會突然多出 AI Providers 和 AI Natives 這兩個角色。它們不是憑空出現的，而是站在 OpenClaw、NeMo 這些開源平台的肩膀上快速成形。上個月我們分析了 SaaS 護城河的三個關鍵變數——資料獨特性、Know-how 深度與切換成本。但在 GTC 之後，必須再加一層：你的護城河是在對的層次上建立的嗎？

這個對比說明了一件重要的事：AI Natives 雖然靈活，但它們缺的正是既有 SaaS 最有價值的東西——多年累積的領域資料和客戶信任。而既有 SaaS 最大的機會，也恰恰在這裡。
⠀⠀⠀
▐  台灣新創現在應該做三個決策

1. 盡快建立 MCP 介面，而不是等待。
圖四的 Agent 架構圖裡，Tools 那格明確標示了 CLI 和 MCP（Model Context Protocol）。這張圖說的其實是一個選擇題：Agent 要跟你的產品互動，有兩條路——走 Tools\/MCP，你主動開放介面，Agent 乾淨地呼叫你的功能；走 Computer Use，你沒有開放介面，Agent 直接模擬人類操作你的 UI，就像用滑鼠亂點。走 MCP，你是 Agent 工具箱裡的一員，有被主動選擇的機會。走 Computer Use，你只是被繞過的 UI，隨時可以被換掉。

這張圖裡還有一個值得注意的細節：Sub-agents 可以進一步呼叫擁有專屬技能的 Skills。這意味著未來你的競爭對手不是單一 AI 工具，而是一個可以動態組合專業 sub-agent 的 orchestrator。你的產品如果不能被當成一個可呼叫的「Skill」，就不會出現在 Agent 的選單裡——MCP 解決的是怎麼接，Skills 決定的是被接進去之後你扮演什麼角色。

2. 找出你的「不可替代資料」並且明確定價。
延續上篇的框架：資料獨特性是決定你在新生態裡議價能力的核心。但 GTC 之後，這件事的急迫性更高了——當 AI Natives 帶著更輕盈的架構進場，你手上多年累積的垂直領域資料是你最重要的資產。無論是考慮 API 授權、資料合作，還是直接整合進 Agentic 工作流程，現在就要盤點清楚，而不是等到客戶流失再想。

3. 用成果定價（Per Result）取代座位定價（Per Seat），哪怕只是先在新客戶身上試。
Jensen 在 keynote 裡說，未來每位工程師都會有「Token 預算」，就像薪資的一部分。這意味著客戶對 AI 的付費邏輯正在從「我買了多少席次」轉向「我得到了多少產出」。對台灣新創來說，這不只是定價調整，而是一個重新定義自己與客戶關係的機會——從賣工具變成對成果負責。

SaaS 不會消失，但它在新生態裡的位置正在重新定義。這不是選擇要不要轉型的問題，而是選擇在哪一層、用什麼方式參與這場重組。GTC 給了我們一張新的競技場地圖，現在的問題是：你站在哪裡？
