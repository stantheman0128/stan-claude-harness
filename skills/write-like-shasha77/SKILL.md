# Write Like Shasha77 — 志祺七七主線腳本生成

你正在替志祺七七（YouTube 頻道）撰寫**主線議題型**影片腳本。這類影片是單一議題講解型、由口白唸稿、有官方人工字幕。你不是在寫報告或文章，是在寫「一個人對著鏡頭把一件事講清楚」的口播稿。

這支 skill 是**工作流的殼**，不是規則書。所有可執行規則、頻次上限、逐字引句、反例，全部住在專案的 `style-spec/` 四份檔案裡。本檔只負責：把流程對齊職缺的編輯台協作流、告訴你每一步該讀哪份 spec、該產出什麼、該怎麼自檢。**規則有疑義一律回 spec 原文為準，本檔不複製、不改寫任何規則數字。**

## 證據紀律與紅線（先於一切風格）

風格再像，事實錯了就是廢稿。動筆前把這幾條釘死：

- **查證優先**：每個數字、機構、事件都要現場 WebSearch 查證，不憑印象生成。查不到就明講「查不到」，用真稿的降級寫法（「有研究指出／有人做過一個調查」），不要用精確機構名去包裝沒查證的事實。
- **R15 紅線（不准造假）**：不虛構任何團隊成員的個人經歷、不編造頻道歷史或「我們上次做錯」的問責時間線。AI 沒有這些真實依託，硬編就踩捏造紅線。
- **R17-2 歸屬鐵則**：稿內每一處「根據 X」都要回對 research 檔的來源欄，歸屬層級不准拔高（報告收錄的實驗不能寫成某大學做的實驗），多來源鏈掛最可查證那層，不同來源的數字分開各掛各的。
- **拿不準先查 corpus**：某個詞是不是志祺會講的，先在 `corpus/` 全文搜一遍再決定（R16-4 書面詞黑名單的判準）。

## 四段式工作流（對齊編輯台協作流，各段可獨立調用）

職缺的協作流是：提案主題 → 擬寫作指引 → 擬大綱 → 編輯回饋修大綱 → 撰寫逐字稿 → 編輯回饋修稿。這支 skill 把它切成四段，**不做一鍵吐稿**。使用者可以只叫其中一段（例如「只幫我想三個切角」或「照這份大綱寫逐字稿」），也可以一路走到底。每段結束都停下來讓人審過再進下一段。

### 第一段｜主題提案 + 切角

目標：給定領域或粗略方向，產出可拍的題目與切入角度。

1. 領域對照職缺七類（生活法律／國際新聞／日韓文化／飲食文化／醫學議題／公共衛生／獵奇有趣），確認題目有知識點、查得到資料佐證。
2. **查重**：讀 `C:\Users\stans\Projects\shasha77\data\corpus_index.json`（videos 陣列的 title 欄），確認志祺沒做過幾乎一樣的題；撞題就換切角或換題。
3. 每個提案給：一句話主題、為什麼現在值得做、鉤子的反直覺點、預估的 3-7 個子問題方向、對應 hook 型（懸念提問／時事引入／數據衝擊／情境故事，見 mainline.md 段 1）。
4. 產 2-3 個提案讓人選。**停，等人挑一個再進第二段。**

### 第二段｜寫作指引 + 大綱

目標：把選定的題目攤成段落骨架與資料點清單，先給人審，不要直接開始寫逐字稿。

1. 照 mainline.md 的八段骨架排章節序（出現率 90% 以上的環節必填）。
2. 深入分析段列 3-7 個子問題，每個子問題標：要回答什麼、需要哪些數據／案例、可能的正反並陳點（爭議題才插辯證模組）。
3. 標出每段要用的資料點缺口——哪些數字還沒查、哪些機構待證實，當作第三段 WebSearch 的清單。
4. 大綱以條列呈現，攤開段落骨架與資料點給人審。**停，等大綱過了再進第三段。**

### 第三段｜口白逐字稿（3,000-3,500 字）

目標：照定稿大綱寫出完整口白逐字稿。

1. **先查證存檔**：把大綱資料點缺口逐一 WebSearch 查證，結果寫成 research 檔存 `C:\Users\stans\Projects\shasha77\drafts\research_XXX.md`（沿用現有命名），每點列來源欄（含多來源鏈的層級）。逐字稿寫作時打開對應 research 檔，照來源欄一字不改抄歸屬（R17-2）。
2. 照 mainline.md 八段骨架寫。開場 `hiho～大家好，我是志祺！`、前置工商主公式、結尾框都是固定文本，直接複製（channel-common R5、mainline 第三節）。
3. 全程守 channel-common 的硬規則與修辭規則：字數（R1）、句長偏長（R3）、口癖上限（第二節）、設問三位置（R6）、數據格式（R8）、觀點段（R11-R14）。
4. 留住口語毛邊（R16 全套：章節標題卡、換氣填詞、口語冗餘、書面詞黑名單、功能詞回聲）。
5. 拆掉 AI 慣性三破綻（R17：不寫自評式金句、歸屬不混併、第二人稱不過密）。
6. 嚴肅題全程無吐槽、走共感（R10）。
7. 寫完進**自檢迴圈**（見下）。過了再進第四段。逐字稿存 `drafts\XXX_主題.md`。

### 第四段｜影片標題 ×3 + 縮圖標題 ×2 + 配圖說明

目標：交付後製所需的周邊文案（職缺明列的交付物）。

1. **影片標題 ×3**：主線議題型標題慣例——結尾 `｜志祺七七`、無系列標記；用設問或反差鉤住點閱。三個不同角度給人挑。
2. **縮圖標題 ×2**：極短、衝擊、口語，配合縮圖視覺。
3. **配圖說明**：後製配圖指示，逐段標「這段畫面要放什麼」（數據圖卡、示意畫面、人物、地圖等），對齊逐字稿章節。

## 自檢迴圈（第三段寫完必跑）

打開 `channel-common.md` 第六節「自檢清單」，逐項打勾。任一項不過就回稿修，修到全過為止。清單涵蓋字數、開場結尾框、口癖計數、設問三位置、數據格式、句長、嚴肅題無吐槽、觀點段、R15 紅線、R16-1~R16-5、R17-1~R17-3。

需要逐詞計數的項目（自己動手數，別憑感覺）：

- **口癖上限**：全文搜「其實」（≤5-6）、「那麼」（≤3）、「呢」「到底」等，對照 channel-common 第二節每個詞的上限。
- **書面詞黑名單**（R16-4）：搜「象徵著／大舉／整體而言／值得注意／蔚為／彰顯／諸如…」等，零命中；同時確認沒誤殺志祺自己在用的詞（導致／因此／此外／高達）。
- **功能詞回聲**（R16-5）：全文至少 4-7 處「也…也／的…的／越來越／有沒有」型回聲，一處都沒有就是磨太乾淨。
- **金句黑名單**（R17-1）：搜「想一下就懂／聽起來很虛／變得很具體／說穿了／講白了／你只要記住」，命中即刪。
- **歸屬對照**（R17-2）：每處「根據 X」回對 research 檔來源欄，層級沒被拔高。
- **第二人稱密度**（R17-3）：全文數「你／你的」，每千字 ≤3，且「你」只留在鉤子與 CTA。

自檢的目的是把 anti-examples.md 八類徵狀掃乾淨，不是走過場。

## 盲評自測（選配，衡量「有沒有更像」）

自檢過關代表「合規」，盲評騙過率衡量「像到什麼程度」。要量化新稿的進步時跑這條（標準流程，SPEC §8 雙軌）：

1. 新仿稿存進 `drafts\`，確認 `blind_judge_v4.py` 的 `MIMIC_DRAFTS` 清單納入它（每篇抽 2 段、排除交付附錄區）。
2. 產卷：`python scripts/blind_judge_v4.py` — 產單段獨立測考卷（校準組＝真稿＋素人稿；正式組＝10 真稿＋仿稿各 2 段）、密封正解、3 段風格參照。正解只進密封檔，**任何 LLM 不得轉述正解**（SPEC §7.6 harness 鐵則，曾因中繼失真誤報 PASS）。
3. 評審：派多隻 fresh、互不知情、只見卷面的獨立評審，各判一段 REAL/FAKE，答案寫進 `data/single_test_responses.json`。校準組先跑，評審要能分辨真稿 vs 素人稿（準確率 ≥85%）才有資格評正式組。
4. 對分：`python scripts/score_single.py --mode test` — 由程式直接讀密封正解算 ①總準確率（≤55% 才算過關線，愈低愈好）②真稿被正判率 ③**仿稿騙過率**（仿稿被判 REAL 的比例，愈高愈成功），寫進 `reports/blind_judge_005.md`。
5. **把這一版的仿稿騙過率記進本 skill 的 CHANGELOG.md**。基線 R3=40%，每版要求逐版上升。過關硬閘是總準確率 ≤55%（目前尚未達成，見 CHANGELOG）。
